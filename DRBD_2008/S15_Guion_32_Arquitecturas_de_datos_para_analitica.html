📈 Arquitecturas de Datos para Analítica

Las arquitecturas de datos permiten comprender cómo se gestionan y analizan grandes volúmenes de información en distintos contextos organizacionales. A continuación, se presenta una síntesis visual que compara sus características, usos y ventajas principales.

⚡ OLTP (Online Transaction Processing)

✅ Características

  • Enfocado en operaciones diarias
  • Consultas simples y rápidas (milisegundos)
  • Datos actuales y detallados
  • Estructura normalizada

🎯 Uso principal

Facturación, registros de transacciones, sistemas operativos.

📊 OLAP (Online Analytical Processing)

✅ Características

  • Enfocado en análisis y toma de decisiones
  • Consultas complejas (segundos a minutos)
  • Datos históricos y agregados
  • Estructura multidimensional

🎯 Uso principal

Análisis de tendencias, reportes gerenciales, inteligencia de negocios.

🏛️ Data Warehouse

Base de datos analítica centralizada que integra datos estructurados, históricos y no volátiles.

  • ✅ Datos integrados y consolidados
  • ✅ Solo lectura y análisis
  • ✅ Esquemas: Estrella, Copo de nieve, Constelación
  • 🎯 Uso: Inteligencia de Negocios (BI)

🌊 Data Lake

Repositorio de datos en bruto que soporta todo tipo de datos (estructurados, semi-estructurados y no estructurados).

  • ✅ Esquema aplicado al consultar (schema-on-read)
  • ✅ Alta escalabilidad y bajo costo
  • 🎯 Uso: Ciencia de datos, Machine Learning, Big Data
