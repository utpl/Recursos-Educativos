INSERT — Inserción de Datos

Agrega nuevas filas a una tabla de la base de datos

info

¿Qué hace?

Permite agregar nuevas filas a una tabla. Puede insertar una o múltiples filas en una sola sentencia.

input

Fuentes de datos

Admite valores literales directos o datos provenientes de una subconsulta SELECT.

Sintaxis — Valores directos INSERT INTO nombre_tabla (col1, col2, col3)
VALUES ('valor1', 42, '2024-01-15');
Sintaxis — Desde consulta SELECT INSERT INTO tabla_destino (col1, col2)
SELECT col1, col2 FROM tabla_origen
WHERE condición;
  • check_circlePuede insertar una o múltiples filas en una sola sentencia.
  • check_circlePuede omitir el nombre de columnas si se especifican valores en el mismo orden de la tabla.
  • check_circlePermite insertar datos desde una consulta SELECT de otra tabla.
  • warningLos valores deben respetar las restricciones de columna (NOT NULL, tipo de dato, FOREIGN KEY).

UPDATE — Actualización de Datos

Modifica valores de filas existentes en una tabla

sync_alt

¿Qué hace?

Modifica valores existentes en una o varias filas de una tabla mediante la cláusula SET.

calculate

Cálculos

Los nuevos valores pueden derivarse de operaciones matemáticas, funciones o subconsultas.

warning ¡Atención crítica! La cláusula WHERE es indispensable para delimitar qué filas se actualizan. Sin ella, todos los registros de la tabla serán modificados.
Sintaxis estándar UPDATE nombre_tabla
SET columna1 = nuevo_valor1,
columna2 = nuevo_valor2
WHERE condición; -- ← OBLIGATORIO para no afectar todo
  • check_circleModifica uno o varios registros según la condición WHERE.
  • check_circlePuede actualizar múltiples columnas en la misma sentencia separadas por coma.
  • check_circleLos nuevos valores pueden incluir cálculos u operaciones sobre el valor actual.
  • dangerousSin cláusula WHERE → se modifican TODOS los registros de la tabla.

DELETE — Eliminación de Datos

Elimina filas específicas de una tabla

delete_sweep

¿Qué hace?

Elimina filas de una tabla según la condición WHERE. Permite ROLLBACK posterior a la eliminación.

filter_alt

Subconsultas

Puede emplear subconsultas en la cláusula WHERE para selección avanzada de registros a eliminar.

warning ¡Atención crítica! Sin la cláusula WHERE, el comando DELETE eliminará todos los registros de la tabla, aunque la estructura de la tabla se conserva.
Sintaxis estándar DELETE FROM nombre_tabla
WHERE condición; -- ← define qué registros eliminar
Con subconsulta DELETE FROM pedidos
WHERE cliente_id IN (
  SELECT id FROM clientes
  WHERE activo = 0
);
  • check_circleElimina filas fila por fila, registrando cada eliminación en el log de transacciones.
  • check_circlePermite ROLLBACK para deshacer la operación dentro de una transacción.
  • check_circlePuede incluir subconsultas para selección avanzada de registros.
  • dangerousSin WHERE → elimina todos los datos de la tabla (similar a TRUNCATE pero más lento).

tips_and_updates
Regla práctica: Use DELETE + WHERE cuando necesite eliminar registros específicos con opción de reversión. Use TRUNCATE para vaciar tablas completas de forma rápida cuando no necesite recuperación y no existan FK activas.
