📌 Caso de Estudio
En una institución universitaria se implementa un nuevo sistema centralizado de gestión académica basado en bases de datos. Durante su despliegue, participan diversos perfiles: personal administrativo que registra asistencias, desarrolladores que construyen módulos del sistema, analistas que requieren consultas complejas para informes estratégicos y especialistas que integran herramientas avanzadas de procesamiento de datos.

La dirección necesita definir qué tipo de usuario corresponde a cada rol para asignar interfaces adecuadas y permisos correctos, evitando riesgos de seguridad o uso ineficiente del sistema. La clasificación adecuada permitirá garantizar que cada actor interactúe conforme a sus competencias técnicas y objetivos operativos.

🎯 Es momento de decidir: ¿Cómo proceder ante esta situación?
❓ Pregunta 1 👤
Personal administrativo registra asistencias mediante una aplicación gráfica. ¿Qué tipo de usuario corresponde a este rol?
❓ Pregunta 2 ⚙️
Desarrollador construye módulos del sistema y conecta la aplicación con la base de datos mediante código. ¿Cómo debe clasificarse?
❓ Pregunta 3 📊
Analista ejecuta consultas SQL complejas para obtener información estratégica. ¿Qué tipo de usuario representa este perfil?
