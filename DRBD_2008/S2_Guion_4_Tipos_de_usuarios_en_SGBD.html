📌 Caso de estudio
En una institución universitaria se implementa un nuevo sistema centralizado de gestión académica basado en bases de datos. Durante su despliegue, participan diversos perfiles: personal administrativo que registra asistencias, desarrolladores que construyen módulos del sistema, analistas que requieren consultas complejas para informes estratégicos y especialistas que integran herramientas avanzadas de procesamiento de datos. La dirección necesita definir qué tipo de usuario corresponde a cada rol para asignar interfaces adecuadas y permisos correctos, evitando riesgos de seguridad o uso ineficiente del sistema. La clasificación adecuada permitirá garantizar que cada actor interactúe conforme a sus competencias técnicas y objetivos operativos. Es momento de decidir cómo proceder ante esta situación.
Pregunta 1
¿Qué tipo de usuario corresponde al personal administrativo que registra asistencias mediante una aplicación gráfica?
Pregunta 2
¿Cómo debe clasificarse al profesional que desarrolla módulos y conecta la aplicación con la base de datos mediante código?
Pregunta 3
¿Qué tipo de usuario representa el analista que ejecuta consultas SQL complejas para obtener información estratégica?
