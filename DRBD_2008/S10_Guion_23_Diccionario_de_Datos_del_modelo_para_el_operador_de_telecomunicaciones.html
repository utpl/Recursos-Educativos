🏠 Diccionario de Datos

El presente diccionario de datos documenta la estructura del modelo de base de datos para un operador de telecomunicaciones. Organiza las entidades principales, sus atributos, tipos de datos y las relaciones que permiten gestionar técnicos, órdenes de instalación y zonas de cobertura.

📌 Componentes del modelo:
• Entidades principales: TÉCNICO, ORDEN_INSTALACIÓN, ZONA_COBERTURA
• Especializaciones: TÉCNICO_FIBRA y TÉCNICO_COBRE (herencia)
• Relaciones: atiende (M:N), se_ubica_en (N:1), es_un (herencia)

🔗 Relaciones del Modelo

🔁 atiende
TÉCNICO (M) ⟷ (N) ORDEN_INSTALACIÓN
Un técnico puede atender múltiples órdenes, y una orden puede ser atendida por varios técnicos.
📍 se_ubica_en
ORDEN_INSTALACIÓN (N) ⟶ (1) ZONA_COBERTURA
Cada orden de instalación se ubica en una zona específica, y una zona puede tener muchas órdenes.
👨‍🔧 es_un (Fibra)
TÉCNICO (1) ⟷ (1) TÉCNICO_FIBRA
Especialización: un técnico puede ser especialista en fibra óptica, heredando atributos.
👨‍🔧 es_un (Cobre)
TÉCNICO (1) ⟷ (1) TÉCNICO_COBRE
Especialización: un técnico puede ser especialista en cobre, heredando atributos.
