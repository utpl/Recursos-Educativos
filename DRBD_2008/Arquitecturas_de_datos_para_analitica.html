OLTP
Online Transaction Processing

Sistema orientado al registro de operaciones diarias con alta velocidad, garantizando integridad y consistencia en cada transacción.

Características
Operaciones cortas y frecuentes
Respuesta en milisegundos
Datos actuales y detallados
Esquema normalizado (3NF)
Alta concurrencia de usuarios
Propiedades ACID
Atomicidad — todo o nada
Consistencia — estado válido siempre
Aislamiento — transacciones independientes
Durabilidad — cambios persistentes
Casos de uso
Sistemas bancarios y de pagos · Facturación y punto de venta · Reservas de vuelos y hoteles · Gestión de inventario en tiempo real
OLAP
Online Analytical Processing

Sistema diseñado para consultas complejas sobre grandes volúmenes de datos históricos, optimizado para inteligencia de negocios y toma de decisiones.

Características
Consultas complejas y multidimensionales
Respuesta de segundos a minutos
Datos históricos y agregados
Estructura desnormalizada
Optimizado para lectura
Operaciones clave
Drill-down — mayor detalle
Roll-up — mayor agregación
Slice & dice — filtrado multidimensional
Pivot — rotación de dimensiones
Casos de uso
Análisis de tendencias de ventas · Reportes gerenciales y dashboards · Segmentación de clientes · Forecasting y planificación financiera
Data warehouse
Almacén de datos analítico

Base de datos centralizada que integra datos estructurados de múltiples fuentes operacionales, optimizada para análisis histórico y reporting empresarial.

Propiedades — Inmon
Orientado a temas de negocio
Integrado de múltiples fuentes
No volátil — solo lectura
Variante en el tiempo (histórico)
Esquemas de modelado
Estrella Copo de nieve Constelación
La capa ETL transforma y limpia los datos antes de ingresarlos al almacén.
Flujo de datos
Fuentes OLTP → ETL / ELT → Staging area → Data warehouse → Herramientas BI
Data lake
Repositorio de datos en bruto

Almacén masivo y flexible que acepta cualquier tipo de dato en su formato original. El esquema se aplica en el momento de la consulta.

Características técnicas
Tipos de datosEstructurado · Semi · No estructurado
EsquemaSchema-on-read
EscalabilidadHorizontal y masiva
CostoBajo (almacenamiento objeto)
AccesoAPI / Query engines
Vs. data warehouse
EstructuraFlexible vs Rígida
UsuariosCientíficos vs Analistas
ProcesamientoAl consultar vs Al cargar
CalidadRaw vs Curada
Tiempo al insightMayor vs Menor
Casos de uso
Machine learning e IA · Análisis de logs y clickstream · Procesamiento de imágenes, audio y video · Big Data y data science exploratoria
